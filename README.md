# Origami - منصة تحليل البيانات الذكية

منصة تحليل بيانات ذكية متقدمة ذاتية الاستضافة، مصممة للشركات والفرق التي تحتاج إلى تحليل بيانات سريع وآمن محلياً. تجمع Origami بين قوة الذكاء الاصطناعي المحلي وأناقة التصميم لتقديم تجربة استخدام فريدة.

## المميزات الرئيسية

### 🔒 الخصوصية الكاملة
- **محلية بالكامل**: جميع البيانات تبقى على خادمك المحلي
- **بدون تبعيات سحابية**: لا توجد خوادم خارجية أو مراقبة
- **تحكم كامل**: أنت المسؤول الوحيد عن بياناتك

### ⚡ الأداء العالي
- **معالجة فورية**: تحليل البيانات بدون تأخير
- **نماذج محسّنة**: نماذج ذكاء اصطناعي محسّنة للأداء المحلي
- **استهلاك منخفض للموارد**: تشغيل على أجهزة عادية

### 🧠 الذكاء الاصطناعي المتقدم
- **دعم Egytronic_1.0**: النموذج الافتراضي المحسّن
- **مرونة النماذج**: دعم نماذج Ollama المختلفة
- **تحليل ذكي**: توليد رؤى واضحة باللغة الطبيعية

### 📊 لوحات معلومات ديناميكية
- **تصورات بيانية تفاعلية**: رسوم بيانية خطية وأعمدة ودوائر
- **تحديث فوري**: البيانات تتحدث في الوقت الفعلي
- **تصدير سهل**: تصدير التقارير والرسوم البيانية

### 🎯 سهولة الاستخدام
- **واجهة حدسية**: تصميم عصري وسهل الاستخدام
- **بدون تقنيات معقدة**: استيراد وتحليل بنقرة واحدة
- **دعم متعدد اللغات**: العربية والإنجليزية

## المتطلبات

- **Docker و Docker Compose**: للنشر المحلي
- **4GB RAM على الأقل**: للتشغيل السلس
- **10GB مساحة تخزين**: لنماذج Ollama والبيانات

## التثبيت والنشر

### 1. استنساخ المستودع

```bash
git clone https://github.com/yourusername/origami-ai.git
cd origami-ai
```

### 2. تشغيل Docker Compose

```bash
docker-compose up -d
```

هذا سيشغل:
- **MySQL Database**: قاعدة البيانات الرئيسية
- **Ollama**: خادم نماذج الذكاء الاصطناعي المحلي
- **Next.js App**: التطبيق الرئيسي
- **Redis**: الذاكرة المؤقتة (اختياري)

### 3. الوصول للتطبيق

افتح متصفحك وانتقل إلى:
```
http://localhost:3000
```

### 4. تحميل نماذج Ollama

```bash
# الدخول إلى حاوية Ollama
docker exec -it origami-ollama bash

# تحميل النموذج الافتراضي
ollama pull egytronic_1.0

# أو نماذج أخرى
ollama pull mistral
ollama pull neural-chat
```

## الاستخدام

### استيراد البيانات

1. انتقل إلى **استيراد البيانات**
2. اسحب ملف CSV أو Excel أو انقر لاختيار ملف
3. معاينة البيانات والتحقق من صحتها
4. انقر على **تحليل البيانات**

### تحليل البيانات الذكي

1. اذهب إلى **اسأل عن بياناتك**
2. اطرح سؤالاً طبيعياً عن بياناتك
3. احصل على إجابة ذكية من نموذج Egytronic_1.0

### عرض لوحات المعلومات

1. انتقل إلى **لوحات المعلومات**
2. شاهد تصورات بيانية تفاعلية
3. استكشف الاتجاهات والأنماط
4. صدّر التقارير

### إدارة النماذج

1. اذهب إلى **الإعدادات**
2. أضف أو أزل نماذج Ollama
3. اختر النموذج الافتراضي
4. قم بتخصيص الإعدادات

## التقنيات المستخدمة

### الواجهة الأمامية
- **React 19**: مكتبة واجهات المستخدم
- **Tailwind CSS 4**: أنماط عصرية
- **Recharts**: رسوم بيانية تفاعلية
- **shadcn/ui**: مكونات واجهة مستخدم

### الخادم
- **Express 4**: إطار عمل الخادم
- **tRPC 11**: نقل البيانات المكتوب
- **Drizzle ORM**: إدارة قاعدة البيانات

### الذكاء الاصطناعي
- **Ollama**: تشغيل نماذج محلية
- **Egytronic_1.0**: النموذج الافتراضي
- **GGUF Format**: تنسيق النماذج المحسّنة

### البنية التحتية
- **Docker**: حاويات
- **MySQL**: قاعدة البيانات
- **Redis**: الذاكرة المؤقتة

## التكوين

### متغيرات البيئة

أنشئ ملف `.env` في جذر المشروع:

```env
# قاعدة البيانات
DATABASE_URL=mysql://origami_user:origami_pass@db:3306/origami_db

# Ollama
OLLAMA_URL=http://ollama:11434

# الأمان
JWT_SECRET=your_jwt_secret_key_here

# التطبيق
NODE_ENV=production
PORT=3000
```

## الأمان

### أفضل الممارسات

1. **غيّر كلمات المرور الافتراضية**
   - غيّر `MYSQL_ROOT_PASSWORD` في docker-compose.yml
   - غيّر `JWT_SECRET` في ملف .env

2. **استخدم HTTPS في الإنتاج**
   - استخدم Nginx أو Apache كـ reverse proxy
   - أضف شهادة SSL/TLS

3. **قيّد الوصول**
   - استخدم جدار حماية
   - قيّد الوصول إلى المنافذ

4. **نسخ احتياطية منتظمة**
   - احفظ قاعدة البيانات بانتظام
   - احفظ نماذج Ollama

## استكشاف الأخطاء

### المشكلة: لا يمكن الاتصال بـ Ollama

```bash
# تحقق من حالة الحاوية
docker ps | grep ollama

# عرض السجلات
docker logs origami-ollama

# اختبر الاتصال
curl http://localhost:11434/api/tags
```

### المشكلة: قاعدة البيانات لا تعمل

```bash
# تحقق من حالة MySQL
docker logs origami-db

# أعد تشغيل الخدمة
docker-compose restart db
```

### المشكلة: التطبيق بطيء

```bash
# تحقق من استخدام الموارد
docker stats

# زيادة الموارد في docker-compose.yml
# أضف limits و reservations
```
